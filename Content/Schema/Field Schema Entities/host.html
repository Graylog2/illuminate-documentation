Host Fields

field field_type description example_values
host_device keyword Identifier for a device (drive network adapter) connected to a system. \Device\HarddiskVolume2
host_hostname keyword/lowercase NetBIOS or dns hostname. corpdc01, corpdc01.local, lab01.corpdomain.com
host_id keyword Host unique identifier (i.e.. SID for Microsoft).
host_ip ip IPv4 and IPv6 addresses. 10.1.2.3, fe80:5cc3:11:4::2c
host_ipv6 ip IPv6 addresses. fe80:5cc3:11:4::2c
host_mac keyword MAC address of host, colon-delimited and lower case. 02:a1:f9:c2:d5:04
host_reference keyword/lowercase Mapped from host_ip or host_hostname in that order - allows a common field to reference for messages that do not provide both (note: CIDR search will not work against this field). 127.0.0.1, corpdc01, corpdc01.local, lab01.corpdomain.com
host_region keyword Name of region source device is located in. us-east-1
host_type_version keyword Operating sytem version of host.
host_virtfw_hostname keyword/lowercase For firewalls that operate as partitioned services this is the name of the logical device.
host_virtfw_id keyword For firewalls that operate as partitioned services this is the ID value of the logical device.
host_virtfw_uid keyword Unique identifier such as a UUID value representing a virtual host.
host_vm_name keyword Virtual system name (not to be confused with the hostname).
