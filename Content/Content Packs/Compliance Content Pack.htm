[Introduction]

Supported Controls

  • This Pack current supports a subset of NIST 800-53 AC-2 controls (Access Management)

Requirements

  • Graylog Server 6.0 or later
  • Graylog Illuminate 5.1.0 or later
  • Microsoft Windows event logs delivered to Graylog and processed by Illuminate

Hint: This version of the compliance content has only been tested with Windows content. The content in this pack may include logs from sources other than Windows but only the Windows source has been reviewed for completeness.

Using This Technology Pack

GIM Categorization

The The Graylog Compliance content pack uses events that have been categorized using Graylog Information Model (GIM) categorization codes and is not restricted to a specific product.

Lookup Tables

The The Graylog Compliance content pack utilizes lookups that are part of Illuminate Core. These lookups are used by Illuminate Core to apply category data to events having one of the user fields user_name, source_user_name, or target_user_name, creating or updat the fields user_category, source_user_category, or , respectively.

The category values used by the Graylog Compliance content pack are:

  • privileged - applied to accounts that have elevated/administrative privileges
  • temp_account - applied to accounts indicating that they are intended for temporary use
  • emergency_account - applied to accounts indicating that they are intended for emergency use

The category values are assigned via the Illuminate Core lookup table titled "Illuminate:Core;Entity Enrichment;Static Accounts Priority Adapter". Entries can be added to this list with the following steps in Graylog:

  1. Navigate to the Enterprise menu and select Illuminate
  2. Click on the "Customization" tab at the top left of the Illuminate page
  3. Locate the lookup titled "Illuminate:Core;Entity Enrichment;Static Accounts Priority Adapter" and click on the "edit" button to the right
  4. Add the account object name (user or group) in the "account name" column
  5. Add one or more category values to the "category" column separating the values with a pipe character ("|")
  6. click the "Configure value" button to save the changes

Exportable Content

The dashboard "Graylog Compliance:Access Control Dashboard" includes a tab titled "Account Management Control Data" which includes a series of compliance control-focused data table widgets which can be used for exporting data from Graylog for use by other tools. To export data from any of the widgets follow these steps:

  1. Click on the actions drop-down (the down-pointing arrow) on the right of the widget containing the data to export and select "Export"
  2. Customize the export properties and click "Start Download"

The downloaded file will be located in either the default download directory for your browser, or wherever you selected to save the file.
