This preview compliance pack provides targeted visibility into Identification & Authentication (IAC), Network (NET), and Endpoint (END) events that support control requirements shared across NIST SP 800-53 Rev 5, PCI DSS v4.0, and US CMMC 2.0 Level 1. The spotlight includes dashboards and a daily report template with tailored widgets for compliance reporting.

Hint: This is a preview pack. Features are subject to change, and more content is planned for future iterations.

Supported Frameworks

This pack currently supports a subset of controls from the following compliance frameworks:

  • NIST SP 800-53 Rev 5

  • PCI DSS v4.0

  • US CMMC 2.0 Level 1

Requirements

  • Graylog 6.1.6+ with a valid Security license

  • Graylog Illuminate 6.3.0 or later

Using This Content Pack

GIM Categorization

The Graylog Compliance content pack uses events that have been categorized using Graylog Information Model (GIM) categorization codes, and in some cases additional schema-defined fields, and is not restricted to a specific product.

Lookup Tables

The The Graylog Compliance content pack utilizes lookups that are part of Illuminate Core. These lookups are used by Illuminate Core to apply category data to events having one of the user fields user_name, source_user_name, or target_user_name, creating or update the fields user_category, source_user_category, or target_user_category, respectively.

The category values used by the Graylog Compliance content pack are:

  • privileged - applied to accounts that have elevated/administrative privileges
  • temp_account - applied to accounts indicating that they are intended for temporary use
  • emergency_account - applied to accounts indicating that they are intended for emergency use

The category values are assigned via the Illuminate Core lookup table titled "Illuminate:Core;Entity Enrichment;Static Accounts Priority Adapter". Entries can be added to this list with the following steps in Graylog:

  1. Navigate to the Enterprise menu and select Illuminate
  2. Click on the Customization tab at the top left of the Illuminate page
  3. Locate the lookup titled "Illuminate:Core;Entity Enrichment;Static Accounts Priority Adapter" and click on the edit button to the right
  4. Add the account object name (user or group) in the account name column
  5. Add one or more category values to the category column separating the values with a pipe character ("|")
  6. click the Configure value button to save the changes

Exportable Content

The dashboard "Graylog Compliance:Access Control Dashboard" includes a tab titled "Account Management Control Data" which includes a series of compliance control-focused data table widgets which can be used for exporting data from Graylog for use by other tools. To export data from any of the widgets follow these steps:

  1. Click on the actions drop-down (the down-pointing arrow) on the right of the widget containing the data to export and select "Export"
  2. Customize the export properties and click "Start Download"

The downloaded file will be located in either the default download directory for your browser, or wherever you selected to save the file.

Report Generation

The compliance content pack includes the following event definitions:

  1. Graylog Compliance:Access Management - Account Lockout
    • Detect account locked events
  2. Graylog Compliance:User Activity Event
    • Detect the following events: account created, account deleted, account modified, privileges assigned, privileges removed, account renamed, group member added, group member removed, account locked, account disabled, account unlocked, account enabled.

These event definitions can fulfill the notification requirements listed in some controls, in order to utilize this the events must be enabled and a notification must be added to the event definitions.
