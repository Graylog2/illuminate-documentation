The Cisco ASA (Adaptive Security Appliance) is a multipurpose firewall appliance from Cisco and is generally used for packet filtering purposes, but it supports many additional features, such as stateful filtering, application inspection, NAT, DHCP, routing, VPN, etc. This technology pack will process Cisco ASA logs, providing normalization and enrichment of common events of interest.

Firepower is an extension to ASA.

Requirements

  • This content pack supports Cisco ASA up to version 9.x or Cisco Firepower 7.1 and higher.

  • Configure ASA/Firepower to send logs to Graylog (see official ASA documentation: Configure Adaptive Security Appliance (ASA) Syslog) (link for PIX 500)

  • Graylog raw/syslog input (ASA syslog format may be rejected by Graylog due to a non-RFC compliant format. Configure an RFC 5424 output to use a Graylog syslog input. Alternatively, send the logs to a raw Graylog input).

  • Graylog Server with a valid Enterprise license, running Graylog version 5.1.10 or 5.2.3 (or later).

Supported Version(s)

  • Up to version 9.x or Cisco Firepower 7.1 and higher.

Stream Configuration

This technology pack includes one stream:

  • "Illuminate:Cisco Device Messages"

Index Set Configuration

This technology pack includes one index set definition:

  • “Cisco Devices Event Log Messages”

If this index set is already defined, then nothing will be changed. If this index set does not exist, then it will be created with retention settings of a daily rotation and 90 days of retention. These settings can be adjusted as required after installation.

What is Provided

  • Parsing rules to extract Cisco ASA logs into Graylog schema compatible fields.

  • An overview dashboard with 4 tabs. These tabs are ASA Overview, Network, Device Authentication, High Priority Messages.

            

  • A dashboard for IDS/IPS messages. To get these messages, Cisco ASA may require a specific license.
