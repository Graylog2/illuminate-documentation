This content pack is for Bitdefender GravityZone but not for private Bitdefender products. This pack will parse out Bitdefender GravityZone logs. 

Supported Version(s)

  • Tested with GravityZone Version 6.56.0-1 (October 2024), but older versions should work, too.

Stream Configuration

This technology pack includes one stream:

  • "Illuminate:Bitdefender Messages"

Hint: If this stream does not exist prior to the activation of this pack then it will be created and configured to route messages to this stream and the associated index set. There should not be any stream rules configured for this stream.

Index Set Configuration

This technology pack includes one index set definition:

  • “Bitdefender Logs”

Hint: If this index set is already defined, then nothing will be changed. If this index set does not exist, then it will be created with retention settings of a daily rotation and 90 days of retention. These settings can be adjusted as required after installation.

Log Format Examples

The Bitdefender GravityZone content pack should work with ~30 different events/modules.

Requirements

  • Bitdefender GravityZone has to be configured to send CEF formated to the matching Graylog Bitdefender CEF input.

  • Graylog 6.2 or higher is needed to have the GravityZone input.

  • The Bitdefender GravityZone input has to be configured correctly. 

  • The (Graylog) server has to be configured to accept TLS 1.2  

Configure Bitdefender GravityZone input (Graylog side)

Please use the official documentation for the input for futher details.

In short, the `Authorization Header Name` has to be: authorization

The Authorization Header Value be set freely, but write it down.

Note: GravityZone requires a TLS 1.2 connection. Your server must be able to handle that. You may want to use Caddy.
 

Preparing Bitdefender GravityZone to route data to a SIEM

Please use the official documentation for this, e.g. https://www.bitdefender.com/business/support/en/77209-463492-generic-integrations-for-siem-platforms-without-https-listeners.html see section `Configure GravityZone to send messages to the SIEM`.

In short, do the following steps:

- Create an API key within Bitdefender GravityZone (User role -> My Account -> API keys). Use the `` role.

- Encode your API key with base64.

- Prepare the required fields for the API call. You need:

  • A valid server url and port e.g. `https://mydomain_name.dev:5555/bitdefender`. Here the port is 5555. In a local instance or with a static IP, an IP might works, too.
  • The Authorization Header Name (authorization) and Value (your choice)
  • The base64 encoded API key (in the example it starts with 222j....)
  • The service type is `cef`
  • A list of wanted modules / event types.
